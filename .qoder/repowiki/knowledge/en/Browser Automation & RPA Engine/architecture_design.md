- Entry point: `manifest.json` defines a service worker (`js/background.js`), a content script (`js/content.js`) injected into all frames, and a side panel (`sidepanel.html` + `js/sidepanel.js`).
- Background orchestrator (`js/background.js`) maintains global state (status, scenario steps, CSV data, loop config) in memory and persists scenarios/CSVs via `chrome.storage.local`; it exposes a message-based API for recording, playback, CSV/scenario CRUD, and floating UI toggling.
- Content script (`js/content.js`) handles DOM-level concerns: event capture for recording clicks/inputs/scrolls, execution of steps (click, input, scroll) with a smart-wait engine using `MutationObserver`, unique CSS selector generation, and dynamic injection of a draggable floating iframe that loads the sidepanel HTML.
- Sidepanel UI (`sidepanel.html` + `js/sidepanel.js`) provides tabbed controls (Automation, CSV Manager, Scenarios), renders step lists with inline editing and CSV column mapping, manages CSV upload/parsing, and displays logs and error modals; it synchronizes with background via `chrome.runtime.sendMessage` and listens for `STATE_CHANGED` broadcasts.
- Dependency direction: UI and content script depend on background for state and orchestration; background depends on content script for DOM execution; communication is strictly asynchronous messaging with fallbacks for context invalidation.
