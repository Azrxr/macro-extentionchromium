- **Entry Point**: `manifest.json` defines a MV3 extension with a `side_panel` UI (`sidepanel.html`), a service worker background script (`js/background.js`), and a content script (`js/content/webSocketMonitor.js`) running in the MAIN world.
- **UI Layer**: `sidepanel.html` coupled with `js/main.js` (webpacked) provides a tabbed interface for scenario building ('Dirty Hands') and guide browsing. It manages state via `GhostState` and renders step lists, datasets, and execution logs.
- **Execution Engine**: `js/main.js` contains the core `executeSteps` interpreter, which processes a JSON-based DSL supporting actions like `click`, `type`, `loop_start`, `if_start`, and `collectData`. It uses `chrome.scripting.executeScript` to inject logic into the active tab.
- **Communication**: `js/background.js` acts as a message relay, particularly for WebSocket events captured by the content script. The content script (`webSocketMonitor.js`) monkey-patches `window.WebSocket` to log traffic via `chrome.runtime.sendMessage`.
- **Data & Scenarios**: Pre-built automation scenarios are stored as JSON files in `sample_scenarios/`, indexed by `sample_scenarios/index.json`, and loaded dynamically into the UI.
